<a name="introduction"></a>

## Einleitung

Mit einer Prozess-Demo kann der Prozess in einer echten Allisa Plattform getestet werden. Damit kann die Qualität der
Prozesskonfiguration bereits während der Prozessmodellierung geprüft werden. Mit "Demo-Umbebungen" kann der Prozess in
**unterschiedlichen Szenarien und Anwendungsfällen** simuliert werden.

Für jede Demo wird eine **eigene Allisa Plattform mit eigener Datenbank** initialisiert, die nur für den Zeitraum der
Demo existiert. Während der Demo ist man auf dieser Allisa Plattform als "Demo Benutzer" angemeldet. Mit dem Beenden
der Demo wird die Allisa Plattform entfernt.

<a name="start-demo"></a>

## Einleitung

Eine neue Demo wird mit dem "Demo"-Button gestartet.

![Demo](/img/docs/de/demo/demo-button.png)

Man gelangt zur Startübersicht der Demo, bei der eine Demo-Umgebung gewählt werden kann.

![Demo](/img/docs/de/demo/demo-start.png)

Mit dem Starten der Demo wird eine neue Allisa Plattform initialisiert und die Demo-Umgebung implementiert. Es wird zur
Demo weitergeleitet.

![Demo](/img/docs/de/demo/demo-example-1.png)

Die Allisa Plattform wird in der Prozessfabrik eingebunden **(1)**. In dem obigen Screenshot startet die Prozess-Demo mit einer
Initial-Aktion. Sollte die Demo-Umgebung weitere Benutzer angegeben haben, kann in der Leite oben **(2)** der Benutzer
gewechselt werden. Dadurch kann der Prozess in unterschiedlichen Rollen getestet werden (sofern der Benutzer Zugriff auf
die Prozess-Instanz hat). Mit "Zu Regeln & Daten" **(3)** kann zu einer anderen Ansicht gewechselt werden (Teil des "Regeln
& Daten" Bereichs). Mit "Konfiguration" **(4)** kann während einer Demo zur Konfiguration gewechselt werden um
gegebenenfalls bestimmte Konfigurationen nachzuschlagen. **Änderungen an der Prozess-Konfiguration während einer Demo
werden nicht in die laufende Demo übertragen.**
